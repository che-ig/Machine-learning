## 1. Введение
**AWK** — это мощная утилита и язык программирования, предназначенный для обработки структурированного текста (табличных данных, логов, CSV-файлов). Название происходит от первых букв фамилий её создателей: **A**льфред **А**хо, **П**итер **В**айнбергер и **Б**райан **К**ерниган (1977 год).

AWK является стандартом POSIX и присутствует практически в любом дистрибутиве Unix/Linux. В современных системах чаще всего используется **GNU AWK (gawk)** или **mawk**.

Главная философия AWK: *«Читай строку, разбирай на поля, применяй правила, выводи результат»*.

---

## 2. Базовый синтаксис

    awk [опции] 'шаблон { действие }' файл_ввода

*   **шаблон (pattern)** — условие, при котором выполняется действие (может быть регулярным выражением, условием или отсутствовать).
*   **действие (action)** — команды, выполняемые в фигурных скобках `{}`, если шаблон истинен.

Если шаблон не указан, действие выполняется для *каждой* строки. Если действие не указано, по умолчанию выполняется `print $0` (печать всей строки).

---

## 3. Ключевые понятия: Записи и Поля

AWK разбивает входной текст на **записи** (строки) и **поля** (колонки).
*   `$0` — вся текущая запись (строка целиком).
*   `$1`, `$2`, `$3`... — первое, второе, третье поле.
*   `$NF` — последнее поле (NF = Number of Fields).
*   `$(NF-1)` — предпоследнее поле.

---

## 4. Встроенные переменные

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `FS` | Разделитель полей на входе (Field Separator) | Пробел/табуляция |
| `OFS` | Разделитель полей на выходе (Output FS) | Пробел |
| `RS` | Разделитель записей на входе (Record Separator) | Перенос строки `\n` |
| `ORS` | Разделитель записей на выходе (Output RS) | Перенос строки `\n` |
| `NR` | Номер текущей записи (Number of Records, сквозной) | - |
| `NF` | Количество полей в текущей записи | - |
| `FNR` | Номер записи в текущем файле (сбрасывается для каждого файла) | - |
| `FILENAME` | Имя текущего обрабатываемого файла | - |
| `ARGC` | Количество аргументов командной строки | - |
| `ARGV` | Массив аргументов командной строки | - |

---

## 5. Структура программы и Шаблоны

Программа AWK состоит из трех основных частей:

    BEGIN { действия до начала чтения файла }
    
    /шаблон/ { действия для строк, подходящих под шаблон }
    
    END { действия после окончания чтения файла }

**Типы шаблонов:**
1.  **Регулярные выражения:** `/error/` (строки, содержащие "error").
2.  **Условия:** `$3 > 100` (строки, где 3-е поле больше 100).
3.  **Диапазоны:** `/start/,/end/` (все строки от "start" до "end" включительно).
4.  **Логические операторы:** `$1 == "admin" && $3 > 500`

---

## 6. Управляющие конструкции и Операторы

**Условные операторы:**

    if (условие) { ... }
    else if (условие) { ... }
    else { ... }

**Циклы:**

    for (i = 1; i <= NF; i++) { print $i }
    while (условие) { ... }
    do { ... } while (условие)

**Операторы:**
*   Математические: `+`, `-`, `*`, `/`, `%`, `^` (степень).
*   Сравнения: `==`, `!=`, `<`, `>`, `<=`, `>=`.
*   Строковые: `~` (соответствует regex), `!~` (не соответствует).
*   Логические: `&&` (И), `||` (ИЛИ), `!` (НЕ).

## 7. Встроенные функции

### Строковые функции
*   `length(строка)` — длина строки.
*   `substr(строка, начало, [длина])` — подстрока (индексация с 1).
*   `split(строка, массив, [разделитель])` — разбить строку в массив.
*   `sub(regex, замена, [строка])` — заменить *первое* совпадение.
*   `gsub(regex, замена, [строка])` — заменить *все* совпадения.
*   `index(строка, подстрока)` — позиция подстроки (0, если нет).
*   `toupper(строка)` / `tolower(строка)` — регистр.

### Математические функции
*   `int(x)`, `sin(x)`, `cos(x)`, `atan2(y,x)`, `exp(x)`, `log(x)`, `sqrt(x)`, `rand()`, `srand([seed])`.

## 8. Ассоциативные массивы
Массивы в AWK — это хеш-таблицы (словари). Ключом может быть строка.

    massive["key1"] = "value1"
    massive["key2"]++

## 9. Реальные примеры использования (Практикум)

Ниже приведены **только проверенные, рабочие примеры**, которые ежедневно используются системными администраторами и разработчиками.

### Категория 1: Базовая обработка текста и колонок

**1. Вывод конкретных колонок (аналог `cut`)**

    # Вывести имя пользователя и его shell из /etc/passwd
    awk -F: '{print $1, $7}' /etc/passwd
    
    # Изменить разделитель на выходе (OFS)
    awk -F: 'BEGIN{OFS=" | "} {print $1, $7}' /etc/passwd

**2. Нумерация строк (аналог `nl` или `cat -n`)**

    awk '{print NR, $0}' file.txt

**3. Удаление пустых строк**

    awk 'NF > 0' file.txt
    # или
    awk 'NF' file.txt

**4. Вывод строк определенной длины**

    # Строки длиннее 80 символов
    awk 'length($0) > 80' file.txt

**5. Замена текста (аналог `sed`)**

    awk '{gsub(/old_word/, "new_word"); print}' file.txt

### Категория 2: Математика, агрегация и статистика

**6. Подсчет строк (аналог `wc -l`)**

    awk 'END {print NR}' file.txt

**7. Сумма значений в колонке**

    # Сумма всех чисел в первом столбце
    awk '{sum += $1} END {print "Сумма:", sum}' numbers.txt

**8. Вычисление среднего значения**

    awk '{sum += $1; count++} END {print "Среднее:", sum/count}' numbers.txt

**9. Поиск минимального и максимального значения**

    awk 'NR==1 {min=$1; max=$1} 
         {if($1<min) min=$1; if($1>max) max=$1} 
         END {print "Min:", min, "Max:", max}' numbers.txt

**10. Подсчет уникальных строк (аналог `sort | uniq -c`)**

    awk '{count[$0]++} END {for (line in count) print count[line], line}' file.txt

### Категория 3: Анализ логов (Nginx, Apache, Syslog)

**11. Топ IP-адресов из access.log (Nginx/Apache)**

    # Формат лога: IP - - [date] "request" status size
    awk '{count[$1]++} END {for (ip in count) print count[ip], ip}' access.log | sort -rn | head -n 10

**12. Подсчет HTTP статусов (группировка по 5-му полю)**

    awk '{status[$9]++} END {for (s in status) print s, status[s]}' access.log

**13. Поиск ошибок с контекстом (диапазонный шаблон)**

    # Вывести все строки от "ERROR" до следующей пустой строки
    awk '/ERROR/,/^$/' application.log

**14. Извлечение времени и сообщения из syslog**

    # Формат: Jun 17 10:00:00 host service: message
    awk '/kernel/ {print $1, $2, $3, $NF}' /var/log/syslog

### Категория 4: Системное администрирование

**15. Мониторинг процессов (аналог `ps` + фильтрация)**

    # Найти PID и %CPU для процессов python
    ps aux | awk '/[p]ython/ {print $2, $3}'
    # Примечание: [p]ython используется чтобы сам процесс awk не попал в выдачу.

**16. Анализ использования дисков (аналог `df`)**

    # Вывести точки монтирования, где использование > 80%
    df -h | awk 'NR>1 && +$5 > 80 {print $6, $5}'
    # Примечание: +$5 принудительно преобразует строку "85%" в число 85.

**17. Поиск файлов больше определенного размера**

    # Найти файлы больше 100MB (104857600 байт) в выводе ls -lR
    ls -lR | awk '$5 > 104857600 {print $9, $5}'

**18. Анализ сетевых соединений (netstat/ss)**

    # Подсчет количества соединений по статусу (ESTABLISHED, TIME_WAIT и т.д.)
    ss -tan | awk 'NR>1 {count[$1]++} END {for (c in count) print c, count[c]}'

### Категория 5: Продвинутая обработка данных

**19. Дедупликация строк с сохранением порядка**

    # Оставить только первые вхождения уникальных строк
    awk '!seen[$0]++' file.txt

**20. Транспонирование таблицы (строки в столбцы)**

    # Исходник:
    # 1 2 3
    # 4 5 6
    # Результат:
    # 1 4
    # 2 5
    # 3 6
    awk '{for (i=1; i<=NF; i++) a[i]=a[i] ? a[i] " " $i : $i} END {for (i=1; i<=NF; i++) print a[i]}' file.txt

**21. Парсинг CSV с кавычками (Только GNU AWK / gawk)**

    # Стандартный awk не умеет корректно парсить CSV, если внутри полей есть запятые в кавычках.
    # gawk решает это через FPAT (Field Pattern)
    gawk -v FPAT='([^,]*)|("[^"]+")' '{print $1, $3}' data.csv

**22. Объединение двух файлов по ключу (аналог SQL JOIN)**

    # file1.txt: ID Name
    # 1 Alice
    # 2 Bob
    # file2.txt: ID Role
    # 1 Admin
    # 2 User
    
    awk 'NR==FNR {role[$1]=$2; next} {print $0, role[$1]}' file2.txt file1.txt

**23. Форматированный вывод (printf)**

    # Выровнять колонки, округлить числа до 2 знаков
    awk '{printf "%-15s | %10.2f | %5d\n", $1, $2, $3}' data.txt

**24. Генерация последовательности (аналог `seq`)**

    awk 'BEGIN {for (i=1; i<=10; i++) print i}'

**25. Чтение данных из переменных окружения**

    export MY_VAR="Hello_AWK"
    awk 'BEGIN {print ENVIRON["MY_VAR"]}'

## 10. Лучшие практики и важные нюансы

1.  **Одинарные кавычки:** Всегда оборачивайте программу AWK в одинарные кавычки `'...'`, чтобы оболочка (bash) не интерпретировала спецсимволы (например, `$`).
    *   *Плохо:* `awk "{print $1}" file` (bash подставит переменную $1).
    *   *Хорошо:* `awk '{print $1}' file`.

2.  **Передача переменных оболочки в AWK:** Используйте флаг `-v` или передачу после имени файла.

        MY_COL=3
        # Способ 1 (через -v, доступно в BEGIN)
        awk -v col="$MY_COL" '{print $col}' file.txt
        # Способ 2 (через присваивание, НЕ доступно в BEGIN)
        awk '{print $col}' col="$MY_COL" file.txt

3.  **Производительность:** AWK очень быстр, но при обработке гигантских файлов (гигабайты) избегайте частых вызовов `system()` или `getline` внутри циклов. Используйте встроенные функции.

4.  **Экранирование:** Если вам нужно использовать одинарную кавычку внутри программы, обернутой в одинарные кавычки, используйте хитрость: `'\''` (конец строки, экранированный апостроф, начало строки) или передавайте её через переменную: `awk -v sq="'" '{print sq}'`.

5.  **Обработка ошибок:** AWK не имеет встроенного механизма `try-catch`. Для сложной логики с обработкой ошибок лучше использовать Python/Perl, но для 95% задач парсинга текста AWK остается непревзойденным инструментом.




# Структура программы AWK и фигурные скобки

## 1. Почему две пары фигурных скобок

В AWK программа состоит из **блоков**. Каждая пара `{}` — это **один блок** с собственным шаблоном и действием.

В команде:

    awk -v MY_VAR="hello" 'BEGIN {print "В BEGIN: " MY_VAR} {print $0}' file.txt

Два блока:

| Блок | Шаблон | Действие | Когда выполняется |
| :--- | :--- | :--- | :--- |
| `BEGIN {print "В BEGIN: " MY_VAR}` | `BEGIN` (специальный) | Напечатать переменную | **Один раз** до чтения файла |
| `{print $0}` | *(пусто = всегда true)* | Напечатать строку | **Для каждой строки** файла |

То есть это **две отдельные инструкции**, которые awk выполняет в разное время.

---

## 2. Полная структура программы AWK

Программа AWK состоит из **трёх типов блоков**, которые выполняются в строгом порядке:

    BEGIN { действие_1 }      ← Выполняется ОДИН РАЗ до чтения файлов
          ↓
    /шаблон/ { действие_2 }   ← Выполняется для КАЖДОЙ строки, подходящей под шаблон
          ↓
    END { действие_3 }        ← Выполняется ОДИН РАЗ после чтения всех файлов

### В развёрнутом виде:

    BEGIN {
        # Инициализация переменных
        # Вывод заголовков
        # Настройка FS, RS и т.д.
    }
    
    # Блок 1: условие и действие
    /регулярка/ {
        # Действие для строк, содержащих регулярку
    }
    
    # Блок 2: другое условие
    $3 > 100 {
        # Действие для строк, где 3-е поле > 100
    }
    
    # Блок 3: без условия (для всех строк)
    {
        # Действие для каждой строки
    }
    
    END {
        # Итоговые вычисления
        # Вывод результатов
    }

---

## 3. Порядок выполнения (пошагово)

Вот что делает awk, когда вы запускаете программу:

    1. Установить переменные из -v
    2. Выполнить ВСЕ блоки BEGIN (по порядку их написания)
    3. Для каждого файла:
       3.1. Открыть файл
       3.2. Для каждой строки файла:
            a. Разбить строку на поля ($1, $2, ...)
            b. Пройти по ВСЕМ блокам без BEGIN/END по порядку
               - Проверить шаблон каждого блока
               - Если шаблон истинен → выполнить действие
               - Если шаблона нет → выполнить действие (всегда)
            c. Перейти к следующей строке
       3.3. Закрыть файл
    4. Выполнить ВСЕ блоки END (по порядку их написания)

---

## 4. Блоков может быть несколько одного типа

Это важный момент — **одного типа блоков может быть много**:

    awk '
    BEGIN { print "Начало 1" }
    BEGIN { print "Начало 2" }
    
    { print "Строка: " $0 }
    
    END { print "Конец 1" }
    END { print "Конец 2" }
    ' file.txt

**Результат:**

    Начало 1
    Начало 2
    Строка: <содержимое 1-й строки>
    Строка: <содержимое 2-й строки>
    ...
    Конец 1
    Конец 2

Все `BEGIN` выполняются по порядку сверху вниз, все `END` — тоже по порядку.

---

## 5. Практический пример со всеми типами блоков

**Задача:** посчитать сумму чисел в первом столбце и вывести среднее.

**file.txt:**

    10 яблок
    20 груш
    30 слив

**Программа:**

    awk '
    BEGIN {
        sum = 0
        count = 0
        print "=== Начало обработки ==="
    }
    
    {
        sum += $1
        count++
        print "Обрабатываю строку " NR ": " $0
    }
    
    END {
        print "=== Конец обработки ==="
        print "Всего строк: " count
        print "Сумма: " sum
        print "Среднее: " sum/count
    }
    ' file.txt

**Результат:**

    === Начало обработки ===
    Обрабатываю строку 1: 10 яблок
    Обрабатываю строку 2: 20 груш
    Обрабатываю строку 3: 30 слив
    === Конец обработки ===
    Всего строк: 3
    Сумма: 60
    Среднее: 20

## 6. Разные шаблоны — разные блоки

Каждый блок может иметь свой шаблон. awk проверяет **все блоки** для каждой строки:

    awk '
    /error/ { print "ОШИБКА: " $0 }
    /warning/ { print "ПРЕДУПРЕЖДЕНИЕ: " $0 }
    /info/ { print "ИНФО: " $0 }
    { print "ОБЩАЯ СТРОКА: " $0 }
    ' log.txt

Если строка содержит и `error`, и `info`, она **пройдёт через оба блока** и будет напечатана дважды (с разными префиксами).

---

## 7. Специальные шаблоны

Помимо `BEGIN` и `END`, есть ещё:

| Шаблон | Когда срабатывает |
| :--- | :--- |
| `BEGIN` | До чтения первого файла |
| `END` | После чтения последнего файла |
| `BEGINFILE` | Перед каждым файлом (только gawk) |
| `ENDFILE` | После каждого файла (только gawk) |
| `/regex/` | Когда строка соответствует регулярке |
| `выражение` | Когда выражение истинно (например, `$3 > 100`) |
| `шаблон1, шаблон2` | Диапазон строк от первого до второго совпадения |
| *(пусто)* | Всегда (для каждой строки) |

---

## 8. Визуальная схема

    ┌─────────────────────────────────────────┐
    │              ПРОГРАММА AWK              │
    ├─────────────────────────────────────────┤
    │  BEGIN { ... }   ← один раз в начале    │
    │  BEGIN { ... }   ← можно несколько      │
    ├─────────────────────────────────────────┤
    │  /pattern1/ { ... }  ┐                  │
    │  /pattern2/ { ... }  │ для каждой       │
    │  condition  { ... }  │ строки файла     │
    │  { ... }             ┘                  │
    ├─────────────────────────────────────────┤
    │  END { ... }     ← один раз в конце     │
    │  END { ... }     ← можно несколько      │
    └─────────────────────────────────────────┘

## 9. Возвращаясь к вашему вопросу

В команде:

    awk -v MY_VAR="hello" 'BEGIN {print "В BEGIN: " MY_VAR} {print $0}' file.txt

Это **минимальная программа из двух блоков**:
1. `BEGIN { ... }` — выполнится один раз перед файлом
2. `{ ... }` — выполнится для каждой строки

Блока `END` нет — и это нормально. Не все программы нуждаются во всех трёх типах блоков.

**Пример только с основным блоком:**

    awk '{print $1}' file.txt

**Пример только с BEGIN:**

    awk 'BEGIN {print "Привет, мир!"}'

(файл даже не нужен — awk выполнит BEGIN и завершится)

**Пример только с END:**

    awk 'END {print "Всего строк: " NR}' file.txt

---

## Итог

- **Фигурные скобки `{}`** — это границы **блока** (шаблон + действие)
- Программа может содержать **любое количество блоков** любого типа
- Порядок выполнения: **все BEGIN → все основные блоки для каждой строки → все END**
- Блоки без `BEGIN`/`END` выполняются **для каждой строки**, если их шаблон истинен
- Отсутствие блока — это нормально, не все программы нуждаются во всех трёх типах
